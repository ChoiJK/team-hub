# Profile Changelog

프로필 변경 이력. 버전을 올릴 때마다 여기에 기록한다.

## Format

```
### {role/persona/rules} — v{N} ({날짜})
- **변경**: 무엇을 바꿨는지
- **이유**: 왜 바꿨는지 (관찰된 문제)
- **효과**: 적용 후 변화 (나중에 기록)
```

## roles/auditor — v1 (2026-04-13)
- **변경**: 신규 역할 생성. 코드베이스 전체 감사 역할 — 품질/보안/아키텍처 감사, 이슈 리포트, bugfix 태스크 직접 등록
- **이유**: ELT 도그피딩 작업에서 코드베이스 전체 감사 수요 확인. reviewer(diff 리뷰)와 구분되는 전체 감사 역할 필요
- **효과**: (측정 대기)

## roles/qa — v1 (2026-04-13)
- **변경**: 신규 역할 생성. 실행 기반 검증 역할 — 타입체크, 린트, 테스트, 빌드 검증. 실패 시 revision 요청 권한
- **이유**: review → done 사이에 실행 검증 게이트 없음. 코드 리뷰 통과해도 빌드/테스트 실패 가능
- **효과**: (측정 대기)

## roles/ux-researcher — v1 (2026-04-13)
- **변경**: 신규 역할 생성. 태스크 파이프라인 밖 자율 연구 역할 — 사용자 행동, 경쟁사, 학습 심리학 지속 리서치. Brain 축적
- **이유**: 일회성 리서치(researcher)와 구분되는 지속적 도메인 전문가 필요. 제품 방향을 JK와 함께 잡아갈 파트너
- **효과**: (측정 대기)

## roles/ux-strategist — v1 (2026-04-13)
- **변경**: 신규 역할 생성. UX 전략 역할 — 리서치 인사이트를 개선안으로 변환, 디자인 원칙 관리, feature 태스크 생성
- **이유**: ux-researcher의 인사이트가 제품에 반영되려면 중간 변환 레이어 필요. designer(실행)와 구분되는 전략 역할
- **효과**: (측정 대기)

## personas/analyst — v1 (2026-04-13)
- **변경**: 신규 페르소나 생성. 구조적 사고, 패턴 인식, 데이터 기반 판단
- **이유**: 기존 페르소나에 시스템 레벨 구조적 사고 관점 부재. auditor 아키텍처 감사, architect 시스템 설계에 필요
- **효과**: (측정 대기)

## personas/empathist — v1 (2026-04-13)
- **변경**: 신규 페르소나 생성. 사용자 감정/맥락 중시, 경험 관점 사고
- **이유**: comfy-ux는 구현 관점("편하게 만들기"), empathist는 이해 관점("왜 그렇게 느끼는가"). ux-strategist/researcher에 필요
- **효과**: (측정 대기)
